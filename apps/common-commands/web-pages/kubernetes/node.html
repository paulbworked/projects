Node: A machine serving as worker in Kubernetes

At its core, Kubernetes is a distributed system which means it manages a containerized environment over one or more machines, known as nodes. So a node is essentially a virtual machines (VMs) where the applications packaged into containers (referred to as Pods) reside. These nodes reside within Azure's virtual networks. Therefore the underlying hardware is that of Azure's datacenters. AKS supports various VM sizes, allowing us to choose the appropriate hardware based on the workload requirements (e.g., CPU, memory, GPU). We have control over the size and type of VMs used for nodes. And Node pools can be configured to use different VM sizes, enabling us to optimize resource utilization for different applications. Fundamentally the node is the virtual machine where containers (Pods) reside, and if there are multiple containers (Pods) working on a node, then at some point that node will have performance issues, whether that be CPU or memory.

Get details of the nodes running:

kubectl get nodes
kubectl get nodes --watch

Get more details of nodes:

kubectl describe no
