Pod Commands

Pod: Deployable unit, representing one or more containers (a self sufficient software package containing all components to run an application)

List all pods across all namespaces:

kubectl get pod --all-namespaces

Get details of a specific pod:

kubectl get po -n datadog
kubectl get pods --namespace=datadog
kubectl get pods --namespace=prod-eu-opra
kubectl get pods -n opra-clienttest -L opra.tritrials.com/version

Delete pod in namespace (which will restart the pod):

kubectl delete pods datadgog-agent-xxss -n datadog

Command to find out how many processors a pod is using:

kubectl get pod -o yaml

In this example:
- The `requests` field specifies the amount of CPU that the container is guaranteed.
- The `limits` field specifies the maximum amount of CPU that the container can use.
In Kubernetes, CPU resources are measured in millicores (m). So, 1200m means 1200 millicores, which is equivalent to 1.2 CPU cores.
1000m = 1 CPU core
1200m = 1.2 CPU cores

```yaml
apiVersion: v1
kind: Pod
metadata:
name: example-pod
spec:
containers:
- name: example-container
image: example-image
resources:
requests:
cpu: "500m" # 0.5 CPU cores
limits:
cpu: "1" # 1 CPU core
```
